# AlphaBot 启动方式说明

## 🚀 推荐方式：一键启动（含 MCP）

### 命令行启动
```bash
# 启动 AlphaBot + MCP Bridge（推荐）
./start-with-mcp.sh

# 停止
./start-with-mcp.sh stop

# 重启
./start-with-mcp.sh restart

# 查看状态
./start-with-mcp.sh status

# 查看日志
./start-with-mcp.sh logs
```

---

## 🖥️ GUI 双击启动

### macOS 应用程序
- **AlphaBot-MCP.app** - 启动 AlphaBot + MCP Bridge（推荐）
- **AlphaBot-Launcher.app** - 仅启动 AlphaBot（不含 MCP）

使用方式：
1. 打开 Finder，进入 `/Users/mima0000/alphabot`
2. 双击 `AlphaBot-MCP.app`
3. 自动启动所有服务并打开浏览器

---

## 🐳 Docker 方式

### 仅 AlphaBot（不含 MCP）
```bash
./start.sh
./stop.sh
```

### 完整命令
```bash
# 启动
docker-compose up -d

# 停止
docker-compose down

# 查看日志
docker-compose logs -f
```

---

## 🌐 访问地址

启动后，在浏览器打开：
- **前端界面**: http://localhost:3000
- **后端 API 文档**: http://localhost:8000/api/v1/docs

---

## 🔌 MCP Bridge 信息

- **MCP 地址**: http://127.0.0.1:8765
- **日志文件**: `/tmp/mcporter-bridge.log`
- **PID 文件**: `/tmp/mcporter-bridge.pid`

---

## ⚠️ 前提条件

1. 安装 Docker Desktop
2. 配置好 `backend/.env` 文件
3. mcporter-bridge 已安装在 `/Users/mima0000/mcporter-bridge`

---

## 🛠️ 故障排查

### MCP 启动失败
```bash
# 查看 MCP 日志
tail -f /tmp/mcporter-bridge.log

# 手动启动 MCP
cd /Users/mima0000/mcporter-bridge
./run-http.sh
```

### 端口冲突
如果 8765 端口被占用，修改 `start-with-mcp.sh` 中的 `MCP_PORT` 变量。

### Docker 未运行
确保 Docker Desktop 已启动。
